Tiers 1–2
Deterministic Stream Resolution
metadata match → mapping rules → resolved or deferred to CID
Tier 1 · Smart Ingestion
Metadata Match
Stream ID present in normalised interaction metadata. If active and live: assign. If inactive: check trust_inactive_metadata config — yes: assign, no: fall to Tier 2.
Tier 2 · Smart Ingestion
Mapping Rules
No explicit metadata. Apply configured mapping rules (e.g. queue_id = credit_card → Credit Card Stream). Match → assign with confidence 0.90–1.0. No match → defer to CID.
✓ Resolved (Tiers 1–2)
Set stream_id · stream_assignment_source · confidence_score. Compute resolved_sltv (A × B × C or flat default). Streams & Promotions resolution complete for this interaction.
⚑ Unresolved
stream_id = null · resolved_sltv = null. Deferred to CID (Tiers 3–4). resolved_sltv computed there once stream resolves.
↓ If unresolved, carry forward to CID
Customer Impact Discovery · Tiers 3–4 · Inferential Resolution
Tiers 3–4
Inferential Stream Resolution
LLM inference → linked stream fuzzy match → resolved or ambiguous
Tier 3 · CID
LLM Inference
No metadata, no rules. LLM reads active stream definitions against conversation context. Confidence ≥ threshold and single clean match → assign (source: inference · confidence 0.50–0.95). Ambiguous or below threshold → Tier 4.
Tier 4 · CID
Linked Streams
Fuzzy resolution using open issues and repeat contact history within context_window against customer profile linked streams. Confident match → assign (source: history). No confident match → pick highest confidence candidate with ambiguity_flag = TRUE for audit.
✓ Resolved (Tiers 3–4)
Set stream_id · stream_assignment_source · confidence_score. Compute resolved_sltv here — Streams & Promotions resolution complete.
Unresolvable
stream_id = null · ambiguity_flag = TRUE. Excluded from stream-scoped scoring. No resolved_sltv produced.
↓ Stream resolved · proceed to promotion resolution

Tiers 1–3
Promotion Resolution
metadata · customer profile · fuzzy match → promotion config loaded
Gate If Promotions module is off: promotion_id = null. Base stream settings apply. No risk amplification. No financial subtraction. Skip to Load Stream Config.
Promotions Module On — Three Tiers
Tier 1
Metadata Match
Promotion ID present in interaction metadata. Assign directly. Source: metadata.
Tier 2
Customer Profile
No metadata. Check for active promotion linked to the resolved stream on customer profile. Match → assign. Source: mapping.
Tier 3
Fuzzy Match
No profile promotion. Fuzzy match against active promotions on the resolved stream. Match → assign (source: history). No match → promotion_id = null, base settings apply.
Load Load stream config. Check for promotion_financial_subtraction configuration. If configured: load value and pass to Financial Attribution. If not: base SLTV applies, no financial subtraction.
Promotion context is now locked. Risk amplification and financial subtraction paths determined by resolved promotion state.
↓ Stream + promotion locked · proceed to financial chain
Financial Chain · Stage 1 & 2 · SLTV → CLTV
Stage 1 & 2
SLTV & CLTV Resolution
resolved_sltv → resolved_cltv via three-path cascade
Stage 1 Resolve SLTV: resolved_sltv = A × B × C or flat default. Always stored on the interaction record, even when Path A resolves CLTV directly from metadata. Source of truth for stream-level financial value.
Stage 2 — CLTV Resolution (Three Paths)
Path A
CLTV from Metadata
customer_cltv present in metadata → use directly. cltv_resolution_source = metadata. resolved_sltv stored but not consumed in this path.
Path B · Tier 1
Customer Metrics
All 3 customer metrics present. resolved_cltv = A × B × C. cltv_resolution_source = customer_metrics.
Path B · Tier 2
Blended Metrics
Partial customer metrics. Blend: customer → stream → org fallback. cltv_resolution_source = blended_metrics.
Path B · Tier 3
Stream Average
No customer metrics. resolved_cltv = resolved_sltv. cltv_resolution_source = stream_average.
↓ resolved_cltv produced · proceed to Impact Scoring
Impact Scoring · Stage 3 · BEI → revenue_at_risk
Stage 3
Impact Scoring
BEI from all platform signals · promotion amplification · revenue_at_risk
Compute BEI calculated from all platform signals: CFS · escalation · duration · urgency · sentiment. No promotion interference at this stage.
Promotion resolved
Amplify BEI: amplified_BEI = BEI × promotion_risk_modifier
promotion_risk_applied = TRUE
revenue_at_risk = resolved_cltv × f(amplified_BEI)
No promotion
BEI unchanged
revenue_at_risk = resolved_cltv × f(BEI)
